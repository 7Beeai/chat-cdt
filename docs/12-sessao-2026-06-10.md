## 1. FIX: lista inteira mostrava "Sem mensagens ainda"

O layout buscava os previews com `.in('conversation_id', ids)` — com 600+
conversas no working set a query string passa de 20KB e o PostgREST devolve
`Bad Request`. O erro era engolido (`console.error` + lista renderiza sem
preview), então TODAS as linhas caíam no fallback "Sem mensagens ainda".

Fix: RPC `chat_conversation_previews(uuid[])` (POST, sem limite de URL) que
devolve exatamente a última mensagem por conversa via LATERAL. De quebra
corrige um segundo defeito do método antigo: o corte global de `ids*4` linhas
ordenadas por data perdia o preview de conversas quietas mesmo quando a
request passava.

**Lição (mesma do reconcile-abacate no dashboard, no MESMO dia): `.in()` com
centenas de UUIDs vira URL gigante e morre. Lote grande → RPC via POST.**

## 3. Anexos no composer

- Clipe habilitado (era "Fase 2"). Aceita imagem (5MB), vídeo/áudio (16MB),
  documento pdf/office/txt/csv (25MB) — espelha limites da Cloud API; nginx
  já tinha `client_max_body_size 25m`.
- Rota nova `POST /api/messages/media` (multipart): valida tipo/tamanho e
  janela 24h → upload `/{phone_number_id}/media` → envia por **media id**
  (link exigiria URL pública estável; signed URL nossa expira) → cópia no
  bucket `chat-media` (mesmo path scheme do inbound) → insere em `messages`
  com o payload no mesmo shape do webhook (MediaBubble renderiza sem caso
  especial).
- UX: chip do arquivo acima do textarea, texto digitado vira legenda
  (caption), Enter envia. Bolha otimista em "pendente" até o realtime trazer
  a linha real.
- **Pendente: teste E2E real** (operador logado → enviar imagem/PDF para um
  número de teste e confirmar recebimento + render na thread).

## 4. Templates de régua no histórico da conversa

Os disparos do motor n8n vivem em `message_log` (sem `conversation_id`) e não
apareciam na thread. RPC `chat_cadence_history(uuid)`:

- casa por unidade + `chat_phone_match_key(to_whatsapp)` (índice novo
  `idx_ml_unit_matchkey`, aditivo, mesmo padrão dos idx das migrations 0013/0016);
- dedup por `wa_message_id` contra `messages`;
- corpo resolvido de `template_inventory` (`body_text`, fallback components
  BODY) — variáveis ficam como `{{n}}` (valores reais só existem no n8n);
- viram linhas sintéticas (`id: mlog-<id>`, sent_by `ai`, status real
  sent/delivered/read/failed) mescladas na timeline, somente exibição.

n8n **intocado** — tudo somente leitura.

### Gotcha encontrado
`template_inventory` usa `template_name`, não `name` (primeira versão da
função quebrou em produção com 42703; corrigida e reaplicada em minutos).
